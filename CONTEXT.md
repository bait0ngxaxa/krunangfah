# Krunangfah Domain

ระบบครูนางฟ้าจัดการข้อมูลโรงเรียน ผู้ใช้ ครู นักเรียน และประวัติการดูแลช่วยเหลือนักเรียนในบริบทโรงเรียน

## Language

**ข้อมูลเลิกใช้งาน**:
ข้อมูลโรงเรียนหรือนักเรียนที่ไม่ควรแสดงหรือถูกใช้ใน workflow ปกติแล้ว แต่ยังเก็บประวัติไว้เพื่อการตรวจสอบหรือกู้คืน
_Avoid_: ข้อมูลเก่า, ข้อมูลที่ไม่ต้องการ, archive

**ข้อมูลทดสอบ**:
ข้อมูลโรงเรียนหรือนักเรียนที่สร้างขึ้นเพื่อทดลองระบบ ฝึกใช้งาน หรือทดสอบ workflow และไม่ถือเป็นข้อมูลใช้งานจริง
_Avoid_: mock data, fake data

**ปิดใช้งานนักเรียน**:
การทำให้นักเรียนรายคนออกจาก workflow ปกติด้วยเหตุผลด้านการจัดการข้อมูล เช่น เป็นข้อมูลทดสอบ ข้อมูลซ้ำ หรือข้อมูลผิด โดยไม่สื่อว่านักเรียนลาออกหรือย้ายโรงเรียน
_Avoid_: ลาออก, ย้ายออก, เรียนจบ, ลบนักเรียนชั่วคราว

**ปิดใช้งานโรงเรียน**:
การทำให้โรงเรียนออกจาก workflow ปกติ โดยผู้ใช้ของโรงเรียนนั้นไม่ควรเข้าใช้งานและข้อมูลนักเรียนไม่ควรถูกนับในภาพรวม แต่ประวัติยังคงอยู่
_Avoid_: ลบโรงเรียนชั่วคราว, ซ่อนโรงเรียน

**สถานะโรงเรียน**:
สถานะที่บอกว่าโรงเรียนอยู่ใน workflow ปกติหรือถูกปิดใช้งาน โดยเป็นแหล่งอ้างอิงหลักสำหรับการอนุญาตให้ผู้ใช้ของโรงเรียนนั้นเข้าใช้งาน
_Avoid_: user status, account status

**กู้คืนโรงเรียน**:
การนำโรงเรียนที่ปิดใช้งานกลับมาอยู่ใน workflow ปกติ พร้อมให้ผู้ใช้และข้อมูลนักเรียนกลับมาใช้งานตามสิทธิ์เดิม
_Avoid_: เปิดโรงเรียน, restore

**เหตุผลการจัดการข้อมูล**:
ข้อความที่ system admin ต้องระบุเมื่อปิดใช้งาน กู้คืน ลบถาวร หรือแก้ไขข้อมูล เพื่ออธิบายเหตุผลทางงานและช่วยตรวจสอบย้อนหลัง
_Avoid_: note, comment

**ประวัติการจัดการข้อมูล**:
บันทึกว่า system admin คนใดทำ action ใดกับโรงเรียนหรือนักเรียน เมื่อใด และด้วยเหตุผลใด
_Avoid_: log, audit log

**เหตุการณ์การจัดการข้อมูล**:
รายการประวัติหนึ่งรายการที่บันทึก action ของ system admin ต่อโรงเรียนหรือนักเรียน พร้อมเหตุผลและ snapshot ที่ทำให้ยังตรวจสอบได้แม้ข้อมูลเป้าหมายถูกลบถาวรแล้ว
_Avoid_: event log row, audit record

**ศูนย์จัดการข้อมูล**:
หน้าหลังบ้านสำหรับ system admin เพื่อค้นหา ตรวจสอบ ปิดใช้งาน กู้คืน mark ข้อมูลทดสอบ และลบถาวรโรงเรียนหรือนักเรียน
_Avoid_: admin data page, data cleanup, old data manager

**ศูนย์ดูแลระบบ**:
หน้าหลังบ้านรวมสำหรับ system admin เพื่อดูแลผู้ใช้งาน โรงเรียน ครู นักเรียน และประวัติการทำงานจากจุดเดียว โดยรวมงานจัดการผู้ใช้งานและศูนย์จัดการข้อมูลไว้ในประสบการณ์เดียวกัน
_Avoid_: แยก user management, admin pages กระจัดกระจาย

**การค้นหาเป็นจุดเริ่มต้น**:
รูปแบบการใช้งานศูนย์ดูแลระบบที่ให้ system admin เริ่มจากการค้นหาโรงเรียน ผู้ใช้ ครู นักเรียน หรือประวัติการทำงาน แล้วค่อยเลือกการจัดการที่เหมาะสมกับผลลัพธ์นั้น
_Avoid_: เริ่มจากฟอร์มแก้ไข, เมนูแยกหลายหน้า

**นักเรียนเป็นศูนย์กลาง**:
รูปแบบการดูและแก้ไขประวัติการดูแลที่เริ่มจากนักเรียนหนึ่งคน แล้วแสดงข้อมูลพื้นฐาน ผลคัดกรอง กิจกรรม ส่งต่อ ให้คำปรึกษา และเยี่ยมบ้านที่เกี่ยวข้องกับนักเรียนคนนั้นในบริบทเดียว
_Avoid_: table-first correction, แก้ record โดยไม่เห็นนักเรียน

**การแก้ไขข้อมูลโดยผู้ดูแลระบบ**:
ความสามารถของ system admin ในการแก้ไขข้อมูลโรงเรียน ผู้ใช้ ครู นักเรียน และประวัติการทำงานเมื่อเจ้าของข้อมูลปกติไม่สามารถแก้ไขเองได้ โดยถือเป็นการดูแลข้อมูลระดับระบบที่ต้องตรวจสอบย้อนหลังได้
_Avoid_: edit everything, database edit, super admin edit

**การแก้ไขข้อมูลแทนครู**:
การที่ system admin เพิ่ม แก้ไข หรือลบข้อมูลการทำงานแทนครูเมื่อครูไม่สามารถจัดการข้อมูลเองได้ โดยยังต้องถือเป็นการแก้ไขข้อมูลโดยผู้ดูแลระบบ
_Avoid_: impersonation, ครูฝากแก้แบบไม่บันทึกเหตุผล

**ครูเจ้าของรายการ**:
ครูที่เป็นเจ้าของหรือผู้เกี่ยวข้องหลักของข้อมูลการทำงาน เช่น ผู้ทำกิจกรรม ผู้บันทึกการเยี่ยมบ้าน หรือครูที่รับดูแลนักเรียน โดยไม่จำเป็นต้องเป็นคนเดียวกับ system admin ที่แก้ไขข้อมูลแทน
_Avoid_: actor, editor, admin owner

**เหตุการณ์การแก้ไขข้อมูล**:
รายการประวัติหนึ่งรายการที่สรุปการแก้ไขข้อมูลโดยผู้ดูแลระบบในหนึ่งครั้ง พร้อมเหตุผล รายการข้อมูลที่เปลี่ยนแปลง ค่าเดิม และค่าใหม่
_Avoid_: field audit row, change log per field

**ผลของการแก้ไขทันที**:
ข้อกำหนดที่ทำให้การแก้ไขข้อมูลโดยผู้ดูแลระบบส่งผลต่อ workflow ปกติและ analytics ทันทีหลังบันทึกสำเร็จ
_Avoid_: รอรอบ sync, cache ค้างหลังแก้

**การแก้ไขสำคัญ**:
การแก้ไขข้อมูลโดยผู้ดูแลระบบที่กระทบสิทธิ์ผู้ใช้ โรงเรียนที่สังกัด analytics ประวัติการดูแล หรือสถานะการจัดการข้อมูล และต้องมี preview ก่อนบันทึก
_Avoid_: แก้เหมือน field ธรรมดา, save ทันทีโดยไม่ตรวจทาน

**ลบรายการย่อยแบบกู้คืนได้**:
การนำข้อมูลประวัติการทำงานรายการหนึ่งออกจาก workflow ปกติและ analytics โดยยังเก็บไว้เพื่อการตรวจสอบย้อนหลังหรือกู้คืน
_Avoid_: hard delete, ลบทิ้งทันที

**ข้อมูลลูกของรายการย่อย**:
ข้อมูลการทำงานที่อ้างอิงรายการประวัติอื่นและต้องถูกจัดการก่อน จึงจะลบรายการต้นทางแบบกู้คืนได้โดยไม่ทำให้ประวัติขาดความสอดคล้อง
_Avoid_: cascade delete, orphan record

**ลบถาวร**:
การลบข้อมูลโรงเรียน นักเรียน ผู้ใช้ และข้อมูลประกอบที่เกี่ยวข้องแบบย้อนกลับไม่ได้ ใช้เฉพาะกรณีข้อมูลทดสอบหรือข้อมูลผิดจริง
_Avoid_: ลบ, เคลียร์ข้อมูล, purge
