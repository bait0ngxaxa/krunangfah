# Use Child Lists For Navigation Not Hidden Actions

เมื่อ system admin เลือกโรงเรียนในศูนย์ดูแลระบบ detail จะแสดง summary และรายการย่อยของผู้ใช้งาน ครู และนักเรียนในโรงเรียนนั้นแบบโหลดเพิ่มได้ แต่ action รายคนต้องเกิดหลังจากคลิกเลือกรายการย่อยให้เป็น selected entity ใหม่ การแยกนี้ทำให้ audit, preview และข้อความยืนยันผูกกับเป้าหมายที่ชัดเจน ลดความเสี่ยงจากการแก้ข้อมูล user/student ผ่านตารางแฝงใน detail ของโรงเรียน
