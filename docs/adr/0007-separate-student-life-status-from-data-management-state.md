# Separate Student Life Status From Data Management State

สถานะชีวิตจริงของนักเรียน เช่น กำลังศึกษา ลาออก ย้ายออก และเรียนจบ ต้องแยกจากสถานะจัดการข้อมูล เช่น ปิดใช้งานหรือข้อมูลทดสอบ ระบบจึงไม่เพิ่มสถานะอย่าง `ARCHIVED` เข้าไปใน `StudentStatus` แต่ใช้ field จัดการข้อมูลแยกบน Student เพื่อไม่ให้ความหมายทางโรงเรียนปนกับการจัดการข้อมูลของ system admin
