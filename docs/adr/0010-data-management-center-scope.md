# Data Management Center Scope

ระบบจะมีศูนย์จัดการข้อมูลสำหรับ system admin เท่านั้น เพื่อจัดการข้อมูลโรงเรียนและนักเรียนที่เป็นข้อมูลเลิกใช้งานหรือข้อมูลทดสอบในหน้าเดียว ขอบเขต phase แรกครอบคลุมการ mark และ unmark ข้อมูลทดสอบ การปิดใช้งาน การกู้คืน และการลบถาวรสำหรับโรงเรียนและนักเรียนรายคน โดยทุก action ต้องมีเหตุผลการจัดการข้อมูล บันทึกเหตุการณ์การจัดการข้อมูล และใช้ preview ก่อนลบถาวรแทนการยิง SQL เอง

**Considered Options**

- ยิง SQL หรือใช้ Prisma Studio เฉพาะกรณี: เร็วแต่เสี่ยงผิดพลาด ไม่มี preview และไม่มีประวัติการจัดการข้อมูล
- ทำเฉพาะลบถาวรโรงเรียน: แก้ pain ตอนนี้ได้ แต่ไม่รองรับกรณีต้องเก็บประวัติไว้หรือกู้คืน
- ทำศูนย์จัดการข้อมูลครบชุด: งานมากกว่า แต่สร้าง workflow ที่ปลอดภัย ตรวจสอบได้ และใช้ซ้ำกับทั้งโรงเรียนและนักเรียน

**Consequences**

- ต้องเพิ่ม schema สำหรับสถานะจัดการข้อมูลและเหตุการณ์การจัดการข้อมูล
- ต้องเพิ่ม auth/query guard เพื่อซ่อนหรือ block ข้อมูลที่ถูกปิดใช้งานจาก workflow ปกติ
- ต้องสร้าง service ที่เป็น SSOT สำหรับ preview, mutation, revalidation และการลบไฟล์จริง
