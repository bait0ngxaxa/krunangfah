# System Operations Center UI Boundary

ระบบจะเพิ่มศูนย์ดูแลระบบเป็นหน้าใหม่สำหรับ system admin เพื่อรวมงานจัดการผู้ใช้งานและงานจัดการข้อมูลไว้ในประสบการณ์เดียว และให้เป็น workflow หลักที่ทำ action ได้จากผลการค้นหาเดียว ไม่พา system admin ออกไปหน้าเดิมจนต้องค้นหาซ้ำ แต่ยังแยก service/action ตาม domain เดิมเพื่อลดความเสี่ยงของ regression ด้านสิทธิ์และ auth หน้าเดิมอย่างศูนย์จัดการข้อมูลและจัดการผู้ใช้งานจะกลายเป็น legacy route หรือ fallback ที่โยงกลับเข้าศูนย์ดูแลระบบ แทนการย้าย logic ทั้งหมดเป็น module เดียวทันที
