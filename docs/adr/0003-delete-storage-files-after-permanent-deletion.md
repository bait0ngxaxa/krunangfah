# Delete Storage Files After Permanent Deletion

การลบถาวรโรงเรียนหรือนักเรียนต้องพยายามลบไฟล์จริงที่อัปโหลดไว้ด้วย โดยระบบจะรวบรวมรายการไฟล์ก่อนลบข้อมูล ลบข้อมูลในฐานข้อมูลด้วย transaction แล้วจึงลบไฟล์จริงหลัง commit หากลบไฟล์บางรายการไม่สำเร็จ ระบบจะบันทึก warning ในประวัติการจัดการข้อมูลแทนการ rollback ฐานข้อมูล เพราะ filesystem หรือ storage ไม่ได้อยู่ใน transaction เดียวกับฐานข้อมูล
