# Keep Searchable Audit Timeline Inside System Center

ประวัติรวมของศูนย์ดูแลระบบจะอยู่ในหน้า `/admin/system` เดิมโดยไม่เพิ่มเมนูใหม่ และแสดงเป็น timeline กลางที่รวมเหตุการณ์การจัดการข้อมูลกับเหตุการณ์การแก้ไขข้อมูลทั้งหมด ผู้ใช้ค้นหาประวัติจากช่องค้นหาเดียวได้โดยไม่กระทบ search entity ของศูนย์ดูแลระบบ

timeline จะโหลดแบบ server-side และแสดงรายการล่าสุด 50 รายการทันทีเมื่อเปิดศูนย์ดูแลระบบ โดยไม่ต้องให้ผู้ใช้ค้นหาก่อน จากนั้นใช้ cursor pagination และมีปุ่มโหลดเพิ่มแทน infinite scroll เพื่อให้ system admin ตรวจสอบย้อนหลังได้แบบตั้งใจและลด query หนักเกินจำเป็น

รอบแรกของ filter จะมีช่องค้นหา ช่วงวันที่ และประเภทเหตุการณ์หลัก ได้แก่ ทั้งหมด การแก้ไขข้อมูล และการจัดการข้อมูล การค้นหาต้องค้นฐานข้อมูลจริง ไม่ใช่กรองเฉพาะรายการที่โหลดมาแล้ว เพื่อไม่ให้สื่อว่าไม่พบประวัติทั้งที่รายการเก่าอยู่หน้าถัดไป

แต่ละรายการใน timeline จะแสดงข้อมูลที่ scan ได้ก่อน ได้แก่ เวลา ประเภทเหตุการณ์ action ภาษาไทย เป้าหมาย ผู้ทำรายการ และเหตุผลแบบสั้น รายละเอียดหนัก เช่น ค่าเดิม/ค่าใหม่ หรือ warnings จะเปิดดูในรายการนั้นแทนการแสดงทั้งหมดตั้งแต่แรก
